What changed:
- Added TelegramOffsetStore and FileTelegramOffsetStore in src/remote-control/telegram.ts.
- TelegramLongPollingTransport now optionally reads a persisted offset before the first getUpdates request.
- After each successfully inspected update, the transport writes the advanced offset to the store. That includes callback updates, normal message updates, and malformed/non-text updates that still advanced update_id.
- Read failures, missing files, or corrupted files fail safe to offset 0, causing at most a one-time backlog replay instead of silently skipping future messages.
- Write failures are reported through the existing transport error callback but do not crash polling.
