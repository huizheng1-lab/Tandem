- 5ffc8c0 D142-1: launch reciprocal executors as job breakaways

Root cause finding:
- The reciprocal executor launcher used plain PowerShell `Start-Process` from `scripts/start-reciprocal-tandem.ps1`.
- A controlled Windows Job Object reproduction confirmed the root mechanism: a normal `Start-Process` child launched inside a job with `JOB_OBJECT_LIMIT_KILL_ON_JOB_CLOSE` was killed when the parent job closed.
- The same reproduction with the fixed executor launcher confirmed the mitigation: Executor B survived after its parent kill-on-close job exited because the launcher now requests `CREATE_BREAKAWAY_FROM_JOB`.
- Current live executor parents also matched this failure shape: both top-level executor processes shared a short-lived launcher parent PID, and that parent was gone while the executors continued.

Event and reliability evidence:
- Application log and Reliability records showed no Tandem.exe crash, no Application Error, and no Windows Error Reporting entry for Tandem during the investigated window.
- Defender Operational log showed routine health reports and a quick scan around 2026-07-18 09:42-09:43 America/New_York, but no remediation/quarantine event for Tandem.
- System log showed Modern Standby/session transitions and Windows Update activity, including an OpenAI.Codex store update failure at 2026-07-18 08:40, but no evidence that Windows Update or Defender directly terminated Tandem.exe.
- That absence of app-crash/security evidence plus the job-object reproduction supports the shared external kill mechanism described in the handoff: parent/job cleanup can silently terminate a whole background process family without Tandem logging an exit.

What changed:
- `scripts/start-reciprocal-tandem.ps1` now launches executor Tandem.exe with a small Win32 `CreateProcessW` wrapper instead of `Start-Process`.
- The wrapper sets `CREATE_BREAKAWAY_FROM_JOB | CREATE_NEW_PROCESS_GROUP` and uses hidden-window startup info.
- The launcher now builds a Windows command line from raw arguments and reports the top-level executor PID.
