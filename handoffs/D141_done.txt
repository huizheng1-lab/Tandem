Likely cause confirmed:
- The previous launcher used a plain hidden PowerShell watchdog process started by `Start-Process`.
- Because the watchdog and server were in ordinary user background process trees, an external process-tree cleanup could kill both without either process reaching its own exit logging path.
- The scheduled task provides an outer service-managed supervisor so a later task tick can revive the watchdog/server pair even after that kind of coordinated kill.
