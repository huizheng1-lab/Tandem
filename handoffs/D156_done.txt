Bound:
- MaxTransitions defaults to 3.
- This caps one invocation to a small chain such as PassiveTest -> prompt next step, with one spare transition for a repaired intermediate state. It is enough for the current 3-step W0016 cadence without creating an unbounded loop across unrelated wishlist items.

Fallback:
- If the helper is missing, the automation endpoint is busy, PassiveTest fails, or any prompt attempt is unavailable, the relay remains in the same durable state the scheduled cron tick already understands.
- Prompt conflicts are recorded as unavailable instead of being retried in a tight loop.
- Existing D133/D149 resume protections are untouched; this helper never retries a failed implementation turn.
