D53-2: handoff-monitor.ps1 fixes

Three real bugs found and fixed:

Bug A (file filter didn't match new convention): the script was using `Get-ChildItem -Filter "HANDOFF_GPT5_D*.md"` only. Handoffs D51+ are named `HANDOFF_D*.md` per `process/LEADER_WORKER_WORKFLOW.md`. As written the script could not see any handoff numbered D51 or higher. Fixed by enumerating both filters and de-duping by full path.

Bug B (regex typo "d+" -> "\d+"): Line 25's git-log handled-detection pattern was `'^D(\d+)-d+[: ]'`. The literal `d+` only matches one or more lowercase 'd' letters, so this regex NEVER matched real commit messages like "D51-1: add /loop...". The git-log fallback was dead code. Fixed to `'^D(\d+)-\d+[: ]'`.

Bug C (defensive): added deduplication (`Sort-Object -Property FullName -Unique`) on the merged handoff list because the same path can never appear twice under these disjoint filters, but it costs nothing and protects against future pattern collisions.

Other improvements during final read-through:
- Added `STATUS:\s*DEFERRED` regex (was `STATUS:s*DEFERRED` - missing backslash, so `s*` matched 0+ 's' literals, which a real "STATUS: " line wouldn't have). Now correctly matches `STATUS: DEFERRED` (with optional whitespace).
- Wrapped `Get-Content $next.Path` in `ErrorAction SilentlyContinue` so a partial read doesn't blow up the script.
- Updated the script header comment to list BOTH naming conventions so future readers don't fall into the same trap.
- Updated `scripts/register-handoff-monitor.ps1`'s Description text to say "(HANDOFF_GPT5_D*.md and HANDOFF_D*.md)" instead of just the old name. The currently-registered Windows Scheduled Task TandemHandoffMonitor was NOT re-registered (description text doesn't affect behavior, and the next manual run picks up the fixed script).
